(static) getResults(options) → {Object|Object|Object}

This function retrieves a list of results from the MONGO_RESULTS_COL collection, owned by options.username
Parameters:
Name Type Description
options Object Contains all the information needed to perform this db operation
Properties
Name Type Description
user Object Contains credential information to connect to the database. If the user doesn't exist, or credentials are invalid, an error will be thrown
username string It is part of options, because it is a route parameter. It is also available under options.user.username.
page number Specify which page of results is requested.
limit number Specify how many items per page is required.
stats string Specify whether we should fetch user result stats instead of a list of results. This is useful to get values like user passrate and user average time.
Source:
  • modules/results-persist.js, line 70
Returns:
  • - If options.stats is undefined - An array with a number of elements equal or less to options.limit. Can be 0. Does not throw an error.
    Type
    Object
  • - If options.stats is undefined. A count field that contains the total count of results - for pagination purposes.
    Type
    Object
  • - If options.stats exists - An object that contains the user username, his passrate and average time.
    Type
    Object
